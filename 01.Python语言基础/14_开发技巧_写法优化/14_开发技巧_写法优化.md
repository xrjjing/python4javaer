# 14_开发技巧_写法优化（总览）

> 本章作为「语言基础」后的补充，不再引入新的语法点，而是把常见的 Python 开发技巧拆成一个个小目录，每个目录都有：讲解文档 + 对应 demo 脚本。  
> 你可以把这里当成自己的「Python 小技巧手册」，在写项目时随时翻看、对照重构。

学习建议：

- 完成 1～13 章基础语法后，再系统学习本章；
- 每学完一个技巧，尝试在项目实战的代码里刻意使用一次；
- 以后在 GitHub / 菜鸟 / StackOverflow 上看到新的好写法，也可以自己按同样结构继续往这里补充。

---

## 目录结构

- `01_字符串格式化_f_string/`  
  使用 f-string 替代字符串拼接，包含基本用法和格式控制 demo。

- `02_推导式_列表字典集合/`  
  列表 / 字典 / 集合推导式，构造新序列的常见写法。

- `03_循环技巧_enumerate_zip/`  
  使用 `enumerate`、`zip` 写更简洁的循环。

- `04_布尔组合_any_all/`  
  使用 `any` / `all` 简化复杂条件判断。

- `05_字典高级用法_get_setdefault_defaultdict/`  
  `get`、`setdefault`、`defaultdict` 在统计、分组场景下的用法。

- `06_pathlib_优雅处理路径/`  
  使用 `pathlib.Path` 代替 `os.path`，在项目实战脚本中统一风格。

- `07_with_上下文管理器与资源释放/`  
  用 `with` 管理文件、数据库会话等需要成对打开/关闭的资源。

- `08_logging_替代print_规范日志/`  
  使用 `logging` 输出可配置、可过滤的日志，为服务与 batch 做准备。

- `09_解包运算符_合并参数与配置/`  
  `*` / `**` 在参数传递、列表/字典合并里的实用场景。

- `10_EAFP_异常优先的编码风格/`  
  「先尝试再捕获」的 EAFP 风格，对比传统「先判断再执行」的 LBYL。

> 每个子目录中都有一个 `xx_技巧名.md` 和一个 `demo_xxx.py`。  
> 建议：当你写完一段代码后，可以打开对应技巧目录，对照看看是否有更 Pythonic 的写法可以使用。
