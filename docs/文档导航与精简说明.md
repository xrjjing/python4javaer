# 文档导航与精简说明

> 目的：去繁从简，统一入口。本文列出“保留核心文档 + 归档文档”清单，并给出阅读顺序。

## 核心必读（保留）
- `README.md`：仓库综述、架构概览、学习路径。
- `docs/architecture.md`：微服务架构与调用流程（Mermaid 图）。
- `docs/快速开始与依赖.md`：环境、启动、依赖、常用脚本。
- `docs/模块总览_接口到实现导航.md`：从前端页面和接口出发，按图索骥找到后端实现（新手友好版总览）。
- `docs/Python学习规划_Java开发者版.md`：学习路线与自检清单。
- `docs/Java_vs_Python_CheatSheet.md`：Java ↔ Python 速查对照。
- `frontend/README.md`：前端页面说明与配置项。
- 各服务 `09_项目说明.md`：服务级别说明（rbac_auth_service 等）。

## 归档/合并说明
- `快速开始指南.md`、`依赖管理指南.md` → 已合并为 `docs/快速开始与依赖.md`。
- `计划.md`、`补充内容总结.md`、`项目完成总结.md` → 信息已在 README 与各服务说明中体现，保留作历史记录；新增待办请写入 `docs/查漏补缺清单.md`。
- `网关_RBAC_后端联调说明.md`：内容较长，作为联调手册保留；快速入口可先看 `docs/快速开始与依赖.md` 的服务启动表。

## 推荐阅读顺序
1. `docs/快速开始与依赖.md`：5 分钟跑通环境与服务。
2. `README.md` + `docs/architecture.md`：理解整体架构与调用链。
3. `docs/Python学习规划_Java开发者版.md`：安排学习节奏。
4. `docs/查漏补缺清单.md`：按缺口补充练习。
5. 需要对比时查 `docs/Java_vs_Python_CheatSheet.md`。

## 维护建议
- 新增文档前先判断：是否可以补充到上述核心文件之一，避免再新增同类文档。
- 历史版本或长篇规划放入 `docs/archive/`（后续需要可创建该目录）。
