# D_系统对接_内部与第三方

> 目标：围绕「系统集成」这个主题，练习用 Python 对接内部 Java 系统和外部网站/服务，包括 HTTP API 调用、简单爬虫等。

本分目录聚合与「系统对接」相关的实战项目：

- `05_简单爬虫_新闻标题抓取`：对接公开 Web 网站；
- `08_系统对接_调用Java服务API`：对接内部 Java 服务。

---

## 阶段 D1：对接公开网站（简单爬虫）

对应项目：`05_简单爬虫_新闻标题抓取`

- 目标：
  - 理解 HTTP 请求基础（headers、超时、状态码检查）；
  - 使用 `BeautifulSoup` 解析 HTML；
  - 练习数据清洗和去重。
- 注意事项：
  - 遵守目标网站的 `robots.txt` 和使用条款；
  - 不要高频请求、不绕过登录与权限控制；
  - 不抓取敏感或个人隐私数据。

---

## 阶段 D2：对接内部 Java 系统（HTTP API）

对应项目：`08_系统对接_调用Java服务API`

- 目标：
  - 封装 Java 服务客户端（base_url、错误处理、超时设置）；
  - 用 FastAPI 编写一层 Python 网关，对外暴露统一风格的接口；
  - 练习将 Java 返回的数据结构「翻译」成更易用的格式。
- 建议根据你公司内部的接口文档，修改：
  - `client_java_service.py` 中的路径和请求体；
  - `api_gateway_example.py` 中的入参/出参模型。

---

## 阶段 D3：综合集成场景（可选）

当 D1 和 D2 都熟练后，你可以尝试设计一个「小型集成系统」：

- 用爬虫从公开网站抓取部分数据；
- 用 Python 网关调用内部 Java 服务补充业务信息；
- 将整合后的数据写入数据库或导出报表；
- 最后再通过 FastAPI 提供查询接口。

这会非常贴近传统「集成层」或「中台服务」的工作模式，只不过你用的是 Python，而核心系统仍然可以继续用 Java 来承载。
