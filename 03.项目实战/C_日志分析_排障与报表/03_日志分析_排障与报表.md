# C_日志分析_排障与报表

> 目标：围绕「日志和监控」这条线，掌握用 Python 快速定位问题、统计错误、生成报表的能力。

本分目录聚合与「日志分析 / 监控」相关的实战项目：

- `02_日志分析与报表生成`；
- `07_监控数据处理_指标聚合与告警`。

---

## 阶段 C1：文本日志分析（错误统计与报表）

对应项目：`02_日志分析与报表生成`

- 目标：
  - 熟悉行级文本处理；
  - 能按日志级别或关键字统计出现次数；
  - 为后续用 pandas 做高级分析打基础。
- 建议练习：
  1. 自己准备 `sample.log`，模仿生产系统的日志格式；
  2. 运行 `log_analyzer.py`，理解按 `[INFO]` / `[ERROR]` 分类计数的逻辑；
  3. 扩展：
     - 按关键字（例如某个 userId）统计行数；
     - 生成简单的 `report.txt`。

---

## 阶段 C2：结构化监控数据分析（CSV + 阈值告警）

对应项目：`07_监控数据处理_指标聚合与告警`

- 目标：
  - 从结构化 CSV 数据中聚合统计；
  - 使用数据类封装统计逻辑；
  - 实现简单的阈值告警规则。
- 建议练习：
  1. 准备 `metrics_sample.csv`，模拟接口耗时等指标；
  2. 运行 `process_metrics.py`，观察统计输出与阈值告警；
  3. 根据你工作中的实际指标（如 QPS、error rate），设计自己的告警规则。

---

## 阶段 C3：从脚本走向服务

当 C1 和 C2 熟练后，可以尝试：

- 用 FastAPI 把「最近一次统计结果」通过 API 提供出去，例如：
  - `GET /metrics/summary` 返回各指标平均值、最大值；
  - `GET /logs/errors/top` 返回错误最多的前 N 类错误；
- 或者将处理结果定期保存为 CSV / JSON，供 BI 工具或其他系统使用。

这条线练熟之后，你会对「用 Python 辅助排查问题和做运维报表」非常有感觉。
