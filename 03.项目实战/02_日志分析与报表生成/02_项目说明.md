# 02_日志分析与报表生成

> 目标：解析一份应用日志或 Web 访问日志，统计不同日志级别或关键字出现次数，练习文件读写、字符串处理和简单统计。

## 功能需求（基础版）

- 从指定日志文件中读取内容；
- 统计每种日志级别（如 `INFO`、`WARN`、`ERROR`）的出现次数；
- 将统计结果打印到终端。

## 使用示例

```bash
python 03.项目实战/02_日志分析与报表生成/log_analyzer.py sample.log
```

## 代码文件说明

- `log_analyzer.py`：日志分析主程序；
- `sample.log`：示例日志文件（需要你自己准备，格式示例见代码注释）。

## 测试说明

- 本项目有配套测试文件：`test_log_analyzer.py`。  
- 在仓库根目录运行下面命令可以只跑本项目的测试：

  ```bash
  pytest 03.项目实战/02_日志分析与报表生成/test_log_analyzer.py
  ```

测试主要验证：

- 给定几行模拟日志时，能按 `INFO`、`WARN`、`ERROR` 等级正确统计数量；
- 当传入的日志文件路径不存在时，会抛出带中文提示信息的 `ValueError`，方便你在脚本调用层面捕获并处理。

## 扩展练习

- 增加命令行参数，统计指定关键字（例如某个用户 ID）出现的次数；
- 将统计结果输出到 `report.txt` 文件；
- 使用 `pandas` 读取结构化日志（如 CSV），做更复杂的统计；
- 将统计结果绘制成简单折线图或柱状图（结合 `matplotlib`）。
